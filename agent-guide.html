Flujo de Ejecución

Esto es lo que sucede paso a paso cuando ejecuta el comando de orquestación. Es como una lista de tareas automatizada.

Automático

🚀 Inicialización

El sistema crea versiones de respaldo y prepara el entorno de trabajo.

Automático

🔍 Fase 1: Análisis Profundo

El agente vb6-analyst escanea todo el código fuente.

Resultado: Inventario completo, métricas de complejidad, y extracción de lógica en formato JSON.

Automático

🗄️ Fase 2a: Migración de Base de Datos

El agente db-migration-architect traduce el esquema legacy a SQLite moderno.

Operaciones:

  • Genera schema.prisma completo
  • Valida integridad referencial
  • Genera y ejecuta seed.ts con datos legacy

Automático

🏗️ Fase 2b: Construcción Backend API

El agente backend-architect genera la API REST sobre el esquema migrado.

Operaciones:

  • Genera DTOs (Tipos de datos)
  • Escribe Controladores y Servicios
  • Documenta API con Swagger

CI Gate

🚦 Puerta de Validación (Build-CI)

El agente build-ci valida que el código generado sea correcto antes de continuar al Frontend.

Comprobaciones:

  • TypeScript sin errores (tsc --noEmit)
  • 0 errores de ESLint
  • Build de producción exitoso

Automático

🎨 Fase 3: Construcción Frontend

El agente angular-architect crea la interfaz de usuario.

Operaciones:

  • Genera componentes Angular Standalone
  • Conecta servicios al backend
  • Implementa rutas y navegación

Auto-Healing

🧪 Fase 4: Verificación y Reparación

El agente testing-verifier entra en un bucle de mejora continua.

Bucle (Max 5 intentos):

  1. Genera Tests Unitarios
  2. Ejecuta Tests
  3. Si falla: Auto-corrige el código y repite

Cómo Crear Nuevos Agentes

El kit es extensible. Puede crear sus propios agentes para tareas específicas siguiendo esta arquitectura de 3 capas.

1. El Orquestador (Workflow)

Es el manual de instrucciones paso a paso. Define qué debe suceder, en qué orden y qué agentes deben intervenir.

workflows/mi-nuevo-flujo.md

2. El Agente (Persona)

Es la identidad de la IA. Aquí le dice "Eres un experto en SQL" o "Eres un diseñador UX". También define qué herramientas tiene permitidas.

agents/mi-nuevo-agente.md

3. La Habilidad (Skill)

Es el conocimiento técnico específico. Contiene guías de estilo, reglas de negocio, tablas de referencia y scripts de ayuda.

skills/mi-nueva-skill/SKILL.md

📝 Paso 1: Definir

Cree un archivo .md en la carpeta agents/. Dele un nombre y un propósito claro.

🧠 Paso 2: Enseñar

Cree una carpeta en skills/ con un SKILL.md. Escriba ahí todo lo que el agente necesita saber.

🚀 Paso 3: Ejecutar

Use el agente en un workflow o llámelo directamente para que aplique su nueva habilidad.
